# SHOP-FE-009: 订单记录和咨询记录页面布局修复

## 任务概述
修复 SHOP-FE-007 导致的布局问题：订单记录和咨询记录页面的标题与下拉选择框应该在一行显示，且下拉框宽度应该自适应（类似添加地址按钮宽度），而不是占据剩余空间。

## 问题描述

从截图可以看到：
1. **订单记录页面** (`pages/account/orders/index.vue`):
   - "订单记录" 标题和 "订单状态" 下拉框不在同一行
   - 下拉框占据了整行宽度（占据了剩余空间）

2. **咨询记录页面** (`pages/account/consultations/index.vue`):
   - "咨询记录" 标题和 "咨询状态" 下拉框不在同一行
   - 下拉框占据了整行宽度

## 期望布局

参考收货地址页面的布局方式：
- 标题和按钮/下拉框在同一行
- 下拉框宽度自适应（类似 "添加地址" 按钮的宽度），不占据剩余空间

```
[订单记录]  [订单状态 ▼]  [右侧空白]
共 8 条订单
```

## 修复清单

### 1. 订单记录页面布局修复
**文件**: `pages/account/orders/index.vue`

- 找到页面头部区域（包含 "订单记录" 标题和 "订单状态" 下拉框的部分）
- 将标题和下拉框放在同一行（使用 flex 布局）
- 下拉框宽度设置为自适应（例如 `w-auto` 或固定宽度如 `w-40`），而不是 `flex-1` 或 `w-full`
- 参考收货地址页面的 "添加地址" 按钮样式

### 2. 咨询记录页面布局修复
**文件**: `pages/account/consultations/index.vue` (或咨询记录页面路径)

- 同样的修复方式：标题和下拉框在同一行
- 下拉框宽度自适应，不占据剩余空间

## 技术要求

1. **布局规范**:
   - 使用 flex 布局：`flex items-center justify-between` 或类似
   - 标题和下拉框在同一行
   - 下拉框宽度参考 "添加地址" 按钮（约 `w-40` 或 `w-auto`）

2. **样式规范**:
   - 保持与现有页面风格一致
   - 使用项目已有的 unocss 类名
